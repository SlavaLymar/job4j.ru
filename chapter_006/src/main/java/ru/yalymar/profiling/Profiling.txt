1. Запускал с параметрами VM "-Xmx1m -XX:+UseSerialGC".
Использовал Visual VM с плагином visualGC.

2. Создавал бесконечное количество объектов - заявок.
На рис. 1 видно как работает малая (minor) и большая (major) сборка мусора.
Малая сборка начинается когда eden заполнен.
Большая сборка выполняется когда в old gen не хватает места.

При малой сборке eden отмечает живые объекты и перемещает их в зону survivor (s0 или s1).
Мертвые объекты удаляются из eden`a.
При повторной малой сборке объекты из s0 передеются в s1. В следующий раз эти объекты из s1 пойдут в old gen.

Большая сборка работает с объектами обоих поколений перенося объекты из eden в survivor и из survivor в old gen.


3. На рис.2 показана ситуация переполнения heap и генерации ошибки - outofmemoryerror.
Запускал с параметрами VM "-Xmx128m -XX:+UseSerialGC".
