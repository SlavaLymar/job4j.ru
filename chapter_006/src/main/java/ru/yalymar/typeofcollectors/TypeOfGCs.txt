1. Запускал с ключами:
-Xmx12m -Xms3m -Xmn1m -XX:+UseSerialGC
Serial GC является сборщиком мусора, который подходит для большинства приложений,
которые не требуют малого времени Stop-the-world (STW).
Он использует только один виртуальный процессор для работы с мусором.
При запуске программы GC не вызывался, видимо из-за того что Eden не заполнился
чтобы выполнять minor gc (малая сборка мусора).

-Xmx12m -Xms3m -Xmn1m -XX:+UseParallelGC
Parallel GC может использовать несколько CPU для ускорения работы приложений.
Его следует использовать, когда требуется обработка относительно больших объемов данных и
длинные паузы (STW) таки приемлемы.
При запуске программы GC вызывался.

-Xmx12m -Xms3m -Xmn1m -XX:+UseParallelOldGC
Тоже самое, что и Parallel GC, но, в отличие от Parallel GC, используется многопоточность в old gen (старом поколении),
а также многопоточность при сжатии old gen (compacting).
При запуске программы GC вызывался.

-Xmx12m -Xms3m -Xmn1m -XX:+UseConcMarkSweepGC
ConcMarkSweep GC следует использовать для приложений, требующих малого времени паузы (STW),
и может совместно использовать ресурсы с выполняемым приложением. Примеры включают приложение
пользовательского интерфейса настольного компьютера, которое реагирует на события, веб-сервер,
отвечающий на запрос, или база данных, отвечающая запросам.
Минусы данного GC - это увеличение потребления ресурсов процессора, снижение общей пропускной способности,
потреблением большей памяти.
При запуске программы GC не вызывался.

-Xmx12m -Xms3m -Xmn1m -XX:+UseG1GC
G1 GC доступен с Java 7 и предназначен для замены CMS GC.
Предназначается для сборки мусора с большим heap (>4 Gb), с небольшим и постоянным временем STW.
Используется для ситуаций когда нужно чтобы время STW было минимальным и постоянным за счет потребление больших ресурсов
процессора.
При запуске программы GC вызывался.
